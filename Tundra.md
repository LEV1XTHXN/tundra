### Заметки:

- Приложение должно быть модульным, я хочу разрабатывать его поэтапно так, чтобы системы было легко делать кохерентными и чтобы можно было масштабировать код и функционал. 
- Я хочу начать с обычных текстовых заметок, которые поддерживают маркдаун, и ещё некоторые UI элементы не из маркдауна. Например, чтобы можно было легко создавать таблицы, добавлять картинки, видео
- При этом важно не перегрузить приложение. В первую очередь оно должно быть быстрым
- Я хочу иметь Graph view как в Obsidian. Такой чтобы было видно все заметки и их ссылки друг на друга
- Приложение будет соответствовать подходу local first, на первое время даже local only. В будущем это должно скалироваться в возможность содержать хранилище на своём хостинге
- Картинки, видео и другие вложения, которые используются в заметках в этом случае будут тоже храниться локально в соответствующих папках/библиотеках
- Я хочу иметь UI похожий на Notion, и формат заметок, который может поддерживать nesting в папках и ссылки заметок друг на друга
- Я хочу чтобы к каждой заметке была возможность создать иконку смайлик или векторный значок. В идеале использовать библиотеку эмодзи из Твиттера/Дискорда
- Было бы хорошо иметь функцию бэкапа, которая сохраняла бы заметки в нужном формате при нажатии соответствующей кнопки в настройках. И этот архив сохранялся на компьютере
- Мне важно чтобы позже при селф хостинге одно хранилище можно было использовать как на телефоне так и на ПК. При этом использовать Conflict-free Replicated Data Type
- Я так же хочу чтобы был отдельный раздел под быстрые заметки, который сохранялся бы как обычная заметка но имел отдельное место в UI. 
- Календарь, в котором можно отмечать события, временные периоды и связывать заметки с конкретными датами и событиями
- Было бы хорошо иметь проверку грамотности с возможностью добавлять в словарь свои слова и термины.
- Иметь локально работающего ИИ агента, который позволяет взаимодействовать с заметками. Например ты его просишь найти во всех заметках информацию связанную с биологией. Или просишь создать заметку о том то том то или создать в календаре приложения событие с конкретным временем. Может даже с голосовым управлением. 
- Домашняя страница где есть ссылки на важные заметки, последние заметки и может виджеты, которые юзер сам пожелает туда вставить

На чём построены аналоги:

Notion - React, Node js, Electron

Obsidian - Electron, TS, markdown

Evernote - React + TS, Electron, Kotlin + Swift

В веб стэках используется DOM - разметка html которая уже позволяет использовать широкий функционал

Backend:
Valkey, MySQL/SQLite

Electron имеет относительно плохой перформанс если быть невнимательным. 
Нужно избежать для поднятия перформанса (документация к Electron):
- Carelessly including modules
- Loading and running code too soon
- Blocking the main process 
- Blocking the renderer process
- Unnecessary polyfills
- Unnecessary or blocking network requests
- Bundle your code
- Call Menu.setApplicationMenu(null) when you do not need a default menu
